# TapSpot 任务游戏系统设计方案

> 版本：v1.0  
> 日期：2026-04-05  
> 状态：设计阶段

---

## 一、核心玩法概述

TapSpot 将从「签到平台」转型为「位置任务游戏」：

```
┌─────────────────────────────────────────────────────────┐
│                    TapSpot 游戏循环                       │
│                                                         │
│   发布任务  →  地图发现  →  到达地点  →  完成任务  →  获得积分  →  排名竞争   │
│      ↑                                                          │
│      └──────────────── 消耗积分解锁更多任务 ──────────────────┘   │
└─────────────────────────────────────────────────────────┘
```

## 二、任务体系设计

### 2.1 任务类型

| 类型 | 图标 | 说明 | 积分范围 | 典型场景 |
|------|------|------|----------|----------|
| **签到任务** | 📍 | 到达指定地点完成签到 | 10-50 | "来天安门签到" |
| **拍照任务** | 📸 | 在地点拍摄照片 | 20-80 | "拍一张三里屯夜景" |
| **问答任务** | ❓ | 回答与地点相关问题 | 30-100 | "故宫午门有几个门洞？" |
| **探索任务** | 🗺️ | 探索新地点（首次访问） | 50-200 | "发现一个新地点" |
| **挑战任务** | 🔥 | 高难度/远距离任务 | 100-500 | "去长城完成签到" |

### 2.2 任务来源

```
┌──────────────────────────────────────────────────┐
│                  任务来源                          │
├────────────────────┬─────────────────────────────┤
│     系统任务         │        用户任务              │
├────────────────────┼─────────────────────────────┤
│ • 每日任务（每日刷新） │ • 付费发布（消耗积分）         │
│ • 节日限定任务        │ • 审核后上线                  │
│ • 城市挑战赛          │ • 奖励金由发布者设定            │
│ • 新手任务（一次性）   │ • 可设置参与人数上限            │
└────────────────────┴─────────────────────────────┘
```

### 2.3 任务属性

```go
type Task struct {
    ID            uint64    // 任务ID
    Title         string    // 任务标题
    Description   string    // 任务描述
    Type          int       // 任务类型：1签到 2拍照 3问答 4探索 5挑战
    
    // 位置信息
    Latitude      float64   // 纬度
    Longitude     float64   // 经度
    LocationName  string    // 地点名称
    Radius        int       // 有效范围（米），默认100m
    
    // 积分与奖励
    Points        int       // 基础积分
    BonusPoints   int       // 额外奖励（如按时完成）
    
    // 限制条件
    MaxParticipants int     // 最大参与人数（0=无限制）
    StartDate      time.Time // 任务开始时间
    EndDate        time.Time // 任务结束时间
    DailyLimit     bool     // 是否每日限制（同一用户不可重复完成）
    
    // 发布者信息
    PublisherType  int      // 1=系统 2=用户
    PublisherID    uint64   // 发布者用户ID
    
    // 状态
    Status        int       // 0=草稿 1=进行中 2=已结束 3=已取消
    
    CreatedAt     time.Time
    UpdatedAt     time.Time
}
```

## 四、积分系统设计

### 4.1 积分类型

| 积分类型 | 说明 | 获取方式 | 消耗方式 |
|----------|------|----------|----------|
| **经验积分** | 永久累计 | 完成任务 | 仅显示，不消耗 |
| **可用积分** | 可消耗 | 完成任务 | 发布任务、兑换奖励 |

> 初期简化为统一积分体系，后续可扩展

### 4.2 积分获取

| 来源 | 积分 | 备注 |
|------|------|------|
| 签到任务 | 10-50 | 根据地点热门程度 |
| 拍照任务 | 20-80 | 需人工审核可选） |
| 问答任务 | 30-100 | 答对才给分 |
| 探索任务 | 50-200 | 首次发现新地点 |
| 挑战任务 | 100-500 | 高难度奖励 |
| 连续签到 | 额外+10% | 连续7天 |
| 每日首杀 | 额外+20 | 每天前3个任务 |

### 4.3 积分消耗

| 用途 | 积分 | 备注 |
|------|------|------|
| 发布任务 | 100-500 | 根据任务类型 |
| 任务加急 | 50 | 置顶1小时 |
| 道具购买 | 20-200 | 暂时加速、双倍积分卡等 |

## 七、任务交互流程

### 7.1 系统每日任务刷新

```
每日 00:00 系统自动执行：
    1. 生成当日每日任务（5-10个）
    2. 重置用户每日任务计数
    3. 更新周榜/月榜统计
    4. 过期任务状态变更为"已结束"
```

### 7.2 用户完成任务流程

```
┌──────────────┐    ┌──────────────┐    ┌──────────────┐    ┌──────────────┐
│  打开地图     │ →  │  查看附近任务  │ →  │  前往任务点   │ →  │  到达范围    │
└──────────────┘    └──────────────┘    └──────────────┘    └──────────────┘
                                                                    ↓
┌──────────────┐    ┌──────────────┐    ┌──────────────┐    ┌──────────────┐
│   积分到账    │ ←  │  任务完成    │ ←  │  提交证明    │ ←  │  触发签到    │
│  更新排名     │    │  记录入库    │    │  位置校验    │    │  进入范围    │
└──────────────┘    └──────────────┘    └──────────────┘    └──────────────┘
```

### 8.2 数据库表

```sql
-- 任务表
CREATE TABLE tasks (
    id BIGINT PRIMARY KEY AUTO_INCREMENT,
    title VARCHAR(255) NOT NULL,
    description TEXT,
    type TINYINT NOT NULL COMMENT '1签到 2拍照 3问答 4探索 5挑战',
    latitude DECIMAL(10, 8) NOT NULL,
    longitude DECIMAL(11, 8) NOT NULL,
    location_name VARCHAR(255),
    radius INT DEFAULT 100,
    points INT NOT NULL DEFAULT 0,
    bonus_points INT DEFAULT 0,
    max_participants INT DEFAULT 0,
    start_date DATETIME,
    end_date DATETIME,
    daily_limit BOOLEAN DEFAULT TRUE,
    publisher_type TINYINT DEFAULT 1 COMMENT '1系统 2用户',
    publisher_id BIGINT,
    status TINYINT DEFAULT 0 COMMENT '0草稿 1进行中 2结束 3取消',
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    updated_at DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
    INDEX idx_status (status),
    INDEX idx_end_date (end_date),
    INDEX idx_location (latitude, longitude)
);

-- 任务完成记录表
CREATE TABLE task_completions (
    id BIGINT PRIMARY KEY AUTO_INCREMENT,
    task_id BIGINT NOT NULL,
    user_id BIGINT NOT NULL,
    completed_at DATETIME NOT NULL,
    points_earned INT NOT NULL,
    proof_photo_url VARCHAR(512),
    answer VARCHAR(255),
    actual_lat DECIMAL(10, 8),
    actual_lng DECIMAL(11, 8),
    distance FLOAT,
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    UNIQUE KEY uk_task_user (task_id, user_id),
    INDEX idx_user_id (user_id),
    INDEX idx_completed_at (completed_at)
);

-- 用户积分表
CREATE TABLE user_points (
    user_id BIGINT PRIMARY KEY,
    total_points INT DEFAULT 0 COMMENT '历史总积分',
    available_points INT DEFAULT 0 COMMENT '可用积分',
    weekly_points INT DEFAULT 0 COMMENT '本周积分',
    monthly_points INT DEFAULT 0 COMMENT '本月积分',
    total_tasks INT DEFAULT 0 COMMENT '完成任务数',
    updated_at DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
    INDEX idx_weekly (weekly_points),
    INDEX idx_monthly (monthly_points)
);

-- 每日任务记录（用于每日刷新控制）
CREATE TABLE daily_task_records (
    id BIGINT PRIMARY KEY AUTO_INCREMENT,
    user_id BIGINT NOT NULL,
    task_date DATE NOT NULL,
    tasks_completed INT DEFAULT 0,
    points_earned INT DEFAULT 0,
    UNIQUE KEY uk_user_date (user_id, task_date),
    INDEX idx_task_date (task_date)
);
```
